Why I used inter vlans
- I implemented it to allow communication between different departments (VLANs) while maintaining network segmentation. I used used Router-on-a-Stick, where a single router interface handles multiple VLANs using subinterfaces.

I configured trunk links  to allow multiple VLAN traffic between switches and the router using IEEE 802.1Q tagging.

## OSPF Configuration (HQ)
I configured OSPF (process ID 1) on the HQ router to enable dynamic routing across the enterprise network.
All internal VLAN networks and WAN links were advertised into OSPF area 0.

This allows:
- Automatic route learning
- Scalable network design
- Simplified management compared to static routing
